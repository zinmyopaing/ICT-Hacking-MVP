Business Model Canvas

Project Title

Smart Study Group Finder

⸻

1. Customer Segments

Who are the main users or customers?
	•	University students
	•	College students
	•	Students looking for study partners
	•	Student clubs and organizations

⸻

2. Value Proposition

What problem does the product solve and what value does it provide?
	•	Helps students find suitable study groups quickly.
	•	Matches students based on subject, schedule, and interests.
	•	Saves time searching for study partners.
	•	Encourages teamwork and better learning.

⸻

3. Channels

How will users find, access, or use the product?
	•	Web application
	•	University website
	•	Social media posts
	•	QR code shared in classrooms

⸻

4. Customer Relationships

How will the product maintain interaction with users?
	•	User registration and login
	•	Notifications for new study groups
	•	Feedback and rating system
	•	Email updates

⸻

5. Revenue Streams / Sustainability Logic

How can the product capture value or be sustained?
	•	Free for students.
	•	Supported by the university.
	•	Future premium features (optional).
	•	Low maintenance cost using cloud services.

⸻

6. Key Resources

What resources are required to build and run the prototype?
	•	Development team
	•	Laptop and internet
	•	Database
	•	Web server
	•	GitHub repository
	•	Programming tools (HTML, CSS, JavaScript, PHP, MySQL)

⸻

7. Key Activities

What main activities must the team or system perform?
	•	Develop the website
	•	Match students with study groups
	•	Store user information securely
	•	Test and fix bugs
	•	Update the system regularly

⸻

8. Key Partners

Who may support, provide data, approve use, or help adoption?
	•	University
	•	Lecturers
	•	Students
	•	IT support team
	•	GitHub for project management

⸻

9. Cost Structure

What costs, time, tools, hosting, or maintenance may be needed?
	•	Web hosting
	•	Internet connection
	•	Development time
	•	Domain name (if needed)
	•	Cloud database
	•	System maintenance

⸻

Final Note

The Smart Study Group Finder MVP allows students to register, create study groups, search for groups, and join groups based on subjects and schedules. The business model supports these features by identifying the target users, required resources, key activities, and partners. This ensures the prototype is useful, practical, and easy for students to use.
