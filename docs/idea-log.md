# Initial IT Venture Idea Log

## Idea 1

Problem Area:
Students forget assignment deadlines.

Target Users:
University students.

Current Alternative:
Using notebooks and reminders.

Proposed IT Solution:
Reminder application.

Possible Technology:
Mobile app.

Why This is Suitable:
Easy to develop as a prototype.

## Idea 2

Problem Area: Students often struggle to find classmates or peers who are studying the same subjects and are interested in forming study groups. As a result, many students study alone and miss opportunities for collaborative learning and knowledge sharing.

Target Users: University students, tutors, and academic support staff.

Current Alternative: Students usually search for study partners through class group chats, social media groups, or personal networks, which can be inefficient and time-consuming.

Proposed IT Solution: A web-based study group finder platform where students can create study groups, join existing groups, search for groups by subject or course, and communicate with other members.

Possible Technology: HTML/CSS/JavaScript prototype, MySQL or Firebase database, user authentication system, search and filter functionality, and group management features.

Why This Is Suitable: The system can be prototyped using web development, database design, UI/UX design, user authentication, and search functionalities.

## Idea 3

Problem Area: Small cafes cannot track inventory.

Target Users: Cafe owners.

Current Alternative: Paper records.

Proposed IT Solution: Inventory Management System.

Possible Technology: Google Sheets Dashboard.

Why This is Suitable: Low-cost prototype.
