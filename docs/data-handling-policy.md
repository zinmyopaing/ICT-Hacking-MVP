Data Handling Policy

## Project
Smart Study Group Finder

---

## Data Collection

The prototype collects only the minimum information required for customer validation and usability testing.

Collected data includes:

- Anonymous Test User ID (e.g., TU-01)
- User Role (Student)
- Prototype Version
- Main Task
- Task Completion Status
- Time Taken (seconds)
- Confusion Point
- Most Useful Feature
- Feedback Score (1–5)
- Interest Level (1–5)
- User Comments
- Requirement Affected
- Device Type

No personal names, phone numbers, email addresses, passwords, or other sensitive personal information are collected.

---

## Data Storage

The validation data is stored as:

- Excel (.xlsx)
- CSV (.csv)
- Power BI Dataset
- GitHub Repository (documentation only)

The prototype does not use an online database or cloud storage for personal information.

---

## Data Access

The collected data can only be accessed by:

- Project Team Members
- ICT105 Instructor (for assessment)
- Teaching Assistants (if required)

The public prototype does not expose any private validation data.

---

## Data Minimization

To protect user privacy, the project intentionally does **not** collect:

- Student Name
- Student ID
- Email Address
- Phone Number
- Password
- Home Address
- Financial Information
- Any Sensitive Personal Data

Only anonymous testing information necessary for usability evaluation is collected.

---

## Responsible Data Rule

The Smart Study Group Finder prototype follows responsible data handling principles by:

- Collecting only necessary information.
- Using anonymous Test User IDs instead of real identities.
- Avoiding unnecessary personal and sensitive data.
- Using collected data only for ICT105 coursework, customer validation, prototype improvement, and final project documentation.
- Not sharing collected data with third parties.
