
# System Requirements

## Project Title

Smart Study Group Finder

---

# Functional Requirements

| Req ID | Requirement | Description |
|--------|-------------|-------------|
| FR-01 | User Registration | Students can register an account. |
| FR-02 | User Login | Students can log in using their account. |
| FR-03 | Create Study Group | Students can create a new study group. |
| FR-04 | Join Study Group | Students can join available study groups. |
| FR-05 | Search Study Groups | Students can search study groups by subject or course. |
| FR-06 | View Group Details | Students can view group name, subject, members, meeting time, and description. |
| FR-07 | Edit or Delete Study Group | Group creators can edit or delete their own study groups. |
| FR-08 | Filter Groups | Students can filter groups by subject or course. |
| FR-09 | Manage Study Groups | Admin can manage or remove inappropriate study groups. |
| FR-10 | Input Validation | The system checks required fields before submission. |
| FR-11 | Confirmation Messages | The system displays success or error messages after each action. |
| FR-12 | Dashboard | Users can view their created and joined study groups. |
| FR-13 | Consistent User Interface | All pages use the same layout, navigation, buttons, and colors. |
| FR-14 | Responsive Design | The website works properly on desktop and mobile devices. |
| FR-15 | Privacy Protection | Only necessary user information is collected and displayed. |
| FR-16 | Requirement Traceability | Every feature supports the project objectives, user stories, and MVP features. |

---

# Non-Functional Requirements

| Req ID | Requirement | Description |
|--------|-------------|-------------|
| NFR-01 | Performance | Pages should load within 3 seconds. |
| NFR-02 | Usability | The system should be easy for university students to use. |
| NFR-03 | Reliability | The system should operate without major errors during normal use. |
| NFR-04 | Security | User passwords should be protected and personal information should not be exposed. |
| NFR-05 | Availability | The system should be accessible whenever students need it. |
| NFR-06 | Compatibility | The website should work correctly on Chrome, Edge, Safari, and mobile browsers. |
| NFR-07 | Maintainability | The system structure should be easy to update and improve. |
| NFR-08 | Scalability | The system should support additional study groups and users in the future. |

---

# Minimum MVP Features

1. Register Account
2. Login
3. Search Study Groups
4. Create Study Group
5. Join Study Group
6. View Study Group Details
7. Dashboard
8. Responsive Interface

---

# Technology Stack

- Frontend: HTML, CSS, JavaScript
- Backend: Firebase or MySQL
- Database: Firebase Firestore / MySQL
- Version Control: GitHub
- Prototype: Web Application
