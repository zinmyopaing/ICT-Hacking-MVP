
# MVP Experiment Plan

## 1. Group and Project Information
- Group name:Group 2 Hacking team
- Project title:Smart Study Group Finder
- Repository link:https://github.com/zinmyopaing/ICT-Hacking-MVP.git
- Main target user:RSU Students especially RIC(international college)
- Prototype platform:

## 2. Experiment Objective
We want to find out whether students can find their desire study groups conveniently and in a short time.

|---|---|---|---|
| FR-01 | Homepage or landing screen | Home page | Yes |
| FR-03 | User input/submission | Report item form | Yes |

## 4. MVP Experiment Type
Select one or combine more than one:
- Clickable prototype
- Landing page test
- Form-based simulation
- Dashboard demo
- Manual service simulation
- Simple web prototype
- Backend/database prototype

Selected experiment type:Simple web prototype

Reason for selection:A simple web prototype lets students use the basic website features.It helps us test whether the website is easy to use and useful before adding more advanced functions.

## 5. Test Users
| Test User Group | Number of Testers | Why They Are Relevant |
|---|---:|---|
| Students | 5 | They are the main target users. |

## 6. Experiment Procedure Summary
1.Ask the selected  5 students to open web prototype.
2.Ask them to creat account and login.
3.Ask them to search for a study group.
4.Ask them to join or creat study group.
5.Observe how they use the website.
6.Request them to fill a short feedback form.
7.Record any problems and suggestions.

## 7. Expected Learning
What decision will your team make after the experiment?
We made some decisions.
1.Check if the students can use the website without difficulty.
2.Find out if the study group search works well.
3.Learn  which features need to be improved.
4.Collect user suggestions for better design and functions.
5.As the final one we decided what changes should be made before developing the full system.
