# MVP Experiment Plan

## 1. Group and Project Information

- Group name: ICT-Hacking-MVP
- Project title: Smart Study Group Finder
- Repository link: [https://github.com/zinmyopaing/ICT-Hacking-MVP](https://github.com/zinmyopaing/ICT-Hacking-MVP.git)
- Main target user: University students who want to find or create study groups.
- Prototype platform: HTML, CSS, JavaScript Web Prototype

---

## 2. Experiment Objective

Our team wants to evaluate whether university students can easily find, create, and join study groups using the Smart Study Group Finder prototype without assistance.

---

## 3. Requirement Scope for the Experiment

| Requirement ID | Requirement Summary | Related Screen/Feature | Tested in This Experiment? |
|---------------|----------------------|------------------------|----------------------------|
| FR-01 | Homepage navigation | Homepage | Yes |
| FR-02 | User Login | Login Page | Yes |
| FR-03 | User Registration | Register Page | Yes |
| FR-04 | Search study groups | Find Groups Page | Yes |
| FR-05 | View group details | Group Details Page | Yes |
| FR-06 | Create a study group | Create Group Page | Yes |
| FR-07 | Join a study group | Join Group Page | Yes |
| FR-08 | Receive notifications | Notification Page | Yes |

---

## 4. MVP Experiment Type

The selected experiment types are:

- Clickable Prototype
- Simple Web Prototype
- Form-based Simulation

### Selected experiment type

Simple Web Prototype with Clickable Prototype

### Reason for selection

Our project has been developed as a functional HTML, CSS, and JavaScript prototype. The prototype allows users to navigate between pages, complete forms, search study groups, create groups, and simulate joining a study group. This experiment is suitable because it enables realistic usability testing before implementing a database or backend system.

---

## 5. Test Users

| Test User Group | Number of Testers | Why They Are Relevant |
|-----------------|-------------------|-----------------------|
| University Students | 5 | They are the primary target users of the Smart Study Group Finder system and frequently participate in group study activities. |

---

## 6. Experiment Procedure Summary

The experiment will be conducted individually with five university students.

Each participant will complete the following tasks:

1. Open the Homepage.
2. Register or log in to the system.
3. Search for a Programming Study Group.
4. Open the Group Details page.
5. Submit a request to join the study group.
6. Create a new study group.
7. Check the Notification page.
8. Complete a feedback questionnaire after testing.

The team will observe the users, record any problems, and collect suggestions for improving the prototype.

---

## 7. Expected Learning

After the experiment, our team expects to learn:

- Whether students can easily understand the user interface.
- Whether students can successfully find and join study groups.
- Which features are most useful to students.
- What usability problems need improvement.
- What additional features users would like to have before developing the final MVP.
