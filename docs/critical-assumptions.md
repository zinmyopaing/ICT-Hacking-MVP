# Critical Assumptions

## Instruction

Identify assumptions that could cause your final prototype to fail if they are wrong.

| Assumption ID | Category | Assumption | Related Requirement / User Story | Risk Level | Current Evidence | How to Test |
|---------------|----------|------------|----------------------------------|------------|------------------|-------------|
| A-01 | User Problem | University students need an easier way to find study partners for their courses. | FR-04 / US-01 | High | Customer interviews from Lab 03 and project research | Ask students to use the Find Groups feature and observe whether they can find a suitable study group. |
| A-02 | Value Proposition | Students believe the Smart Study Group Finder is more convenient than using class chat groups or social media. | FR-04 / US-02 | High | Interview feedback and competitor analysis | Ask users which method they prefer after using the prototype. |
| A-03 | Usability | Users can easily navigate between Homepage, Find Groups, Group Details, Join Group, and Notification pages without instructions. | FR-01 / US-03 | Medium | Prototype developed, but no usability testing yet | Conduct a task-based usability test and observe navigation success. |
| A-04 | Technical Feasibility | The HTML, CSS, and JavaScript prototype works correctly on both desktop and mobile devices. | FR-01 to FR-08 | Medium | Prototype completed and manually tested by the development team | Test the prototype on different screen sizes and browsers. |
| A-05 | Business Logic | Students are willing to create study groups and submit join requests through the system instead of using messaging apps. | FR-06 / FR-07 / US-04 | High | No real user validation yet | Ask testers to create a study group and submit a join request, then collect feedback. |
| A-06 | Data Handling | User information entered in forms (registration, create group, join group) is accurate and sufficient for the study group process. | FR-02 / FR-03 / FR-06 / FR-07 | Medium | Prototype includes validation for required fields | Observe users completing forms and identify missing or incorrect data. |

## Categories

Use these categories:

- User Problem
- Value Proposition
- Usability
- Technical Feasibility
- Business Logic
- Data Handling
